# Weather Generation 说明

这个模块负责从经纬度和时间输入，结合从公共 API 请求到的天气历史记录、海拔与本地计算的日出日落时间，生成某片区域未来一段时间的天气

## 术语

- **Inter Day Weather**: 某一天的天气记录，不存在内部时间更细的记录。也用来作为细化的 Intra Day Weather 的生成前置。
  - **降水量**：当天降水量。其同时决定当天是否无雨/雪或者降雨/雪的程度。
  - **日出/日落时间**：顾名思义
  - **最高/最低气温**：顾名思义。如果是雨雪天气，会相应调整（未来的天气通过自动平均过程会相应抵消）
  - **云量**：顾名思义。如果是雨雪天气，云量会相应增加（未来的天气通过自动平均过程会相应减少）
  - **风速**：顾名思义。如果是雨雪天气，风速会相应增加（未来的天气通过自动平均过程会相应减少）
- **Intra Day Weather**: 一天内部的天气，本质上是一个函数，输入当天的时间就能获取当时的天气，由 Inter Day Weather 生成。
  - 生成过程中不止考虑锚定参数，也会考虑前一天结束时的温度；如果有要求，还会加入修正参数，以令其符合锚定参数描述的平均状况
  - **降水曲线**：实际上按照 4 小时界限（每天 6 区间）分配的降水量。并且根据后续区间随机附加一个“趋势”描述
  - **云量曲线**：4 小时 × 6 的云量，通过降水量给对应区域加权且确保当天总平均值符合前置
  - **风速曲线**：4 小时 × 6 的风速，通过降水量给对应区域加权且确保当天总平均值符合前置
  - **气温曲线**：分为升温段和降温段的温度曲线，分别使用各自的正弦曲线拟合。分隔点由日出和日落时间决定（先算出正午=(日出+日落)/2, 正午+2 是最高点，日出时间为最低点）。然后根据降水量、云量、风速在各区间微调


## 流程

直接输入：经纬度，时间

间接输入（锚定参数）：
- 用 suncalc3 获取日出日落时间
- 从 Open-Elevation API 获取海拔高度
- 从 Open-Meteo API 获取去年此月份的
  - 月平价最高/最低气温
  - 月降水天数
  - 降水日平均降水量
  - 月平均云量
  - 月平均风速
