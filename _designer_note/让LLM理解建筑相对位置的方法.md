
## 极坐标表示法

* 等级3：1km~300m，看见有建筑
  * 建筑：只保留最基本的 building:xxx 和建筑高度/层数，label 也同步替换为 building:xxx，避免透露额外的店名信息
  * POI：完全去除；需要显示的POI已经挂载到建筑中了
  * 线与区域：类似建筑，只保留最基本的类型信息，例如highway:xxx，避免额外透露路名/区域名信息。此外还要过滤掉视野角5°以内的细碎feature，模拟太远太小易被地形遮挡。
* 等级2：300m~100m，辨别建筑物大致类型（商店、住宅...）
  * 建筑：在等级3基础上添加name与brand信息。label 返回到正常的模式。
  * POI：只透露基本信息，例如amenity:waste_basket，其他的不显示
  * 线与区域：类似上一等级，但不会过滤掉视野角5°内的feature。
* 等级1：100m~30m，辨别建筑具体细节（店名等）
  * 透露所有细节。线与区域返回正常的 label 形式。
* 等级0：30m以内，所有细节
  * 用网格表示

### 位置表示

等级0内细网格描绘建筑相对关系（5m的格子宽度）

格子内填写 label，网格下面以 `<label>(id=...): \n<tag1>\n<tag2>...` 的形式跟上格子里填不下的细节
具体形式例子：
```md
McGinnis Ferry Road - highway:secondary (id=way/252545208):
* maxspeed: 45 mph
* oneway: yes
* tiger:county: Gwinnett, GA
...
```
以id为unique entry来展示细节。

等级1~3仅包含极坐标体系下：
* 最近点、最远点、中心点的极坐标
* 最宽的两点的极坐标与所占视野角度

无论是建筑体还是区域，均按照这个方式呈现

每个等级也类似 `<label>: (id=...)\n<坐标信息>\n<tag1>\n<tag2>...` 这样。
具体形式例子：
```md
McGinnis Ferry Road Trail - highway:cycleway:
* (id=way/1480967447)
  * 最近点距离39m / 方位31°，最远点距离967m / 方位307°，中心点距离17m / 方位340°
  * 边界点1距离323m / 方位295°，边界点2距离938m / 方位126°，视野角宽191°
  * maxspeed: 45 mph
  * oneway: yes
  * tiger:county: Gwinnett, GA
* (id=way/463137310)
  * 最近点距离...
...
```
以id为unique entry来展示细节，但 label 相同的会被聚类到一起展示。（这里假设连 label 都能够重合，说明性质都差不多）

即使 label 相同，也会按照距离进行大致分群。每个群内部若要素数量很多，则只会挑其中一些显示，其他都隐藏只构成要素数目。

在排序上，先分两大块，第一块只显示建筑与POI信息，内部等级1~3依次排列；第二块显示线类元素与区域（比如工业区/居住区/林地等），内部等级1~3依次排列。
POI有可能在高等级时所有细节全部遗失，如果这样那么这个POI就不显示了。
