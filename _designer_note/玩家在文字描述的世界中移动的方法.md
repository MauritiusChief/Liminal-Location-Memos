
## 自动从 overpass api 查询数据填充数据库的方法

已有数据库记录查询点。每当玩家的坐标距离最近的查询点超过300米，则触发一次查询，以当前坐标为新的查询点。

## 环境描述

环境描述是可以被存储与更新的。每段环境描述会附带其对应查询点与范围信息，并决定这段环境描述的生效范围。

* 大环境描述：对应环境摘要范围半径500米~1000米（取决于地形空旷与否），生效范围为300米。（换言之，玩家走出300米了就要生成新大环境描述，倘若再回来则又启用这个环境描述）
* 小环境描述：对应环境摘要范围200米，生效范围100米，同时生成/更新时会考虑200米范围内其他小环境描述。

大环境描述与小环境描述的 LLM 系统提示词是不一样的，因为毕竟功能不一样。大环境描述主要描述周遭状况，而小环境描述则记录对具体建筑外形的关键特征、具体环境细节等。
也因此，大环境描述不常更新，而小环境描述常常伴随着玩家的“具体观察”（实际上是 LLM 的自由发挥）而记录文本性质的细节。

## 玩家的移动

玩家的移动需要 LLM 自行推断在极坐标中，玩家向哪个方向移动、移动了多少距离。可以认为玩家的移动归类为以下三类：

* 直述，直接说往哪个方向移动了多远
* 移动到某地，可以参考极坐标环境摘要提示词
* 沿着xx移动，也可以参考此特征的极坐标环境摘要提示词

确定玩家在极坐标中的移动后，LLM 可以 call function，参数是角度与距离

call function 之后，程序根据新的位置先自动判断能否按距离复用大小环境描述，如果不能复用则调用 LLM 进行大小环境描述的生成。生成/复用了大小描述之后，把这些作为系统提示词再放入主对话过程的系统提示词。
一同放入系统提示词的还有环境摘要，以及周遭小描述中被标记为远处可见的部分。
