容器以建筑id、房间id、其他容器id或经纬度的形式记录在存档中。

玩家的衣服裤兜、手上都可以存放少量物品作为“背包”（本质上是一些玩家随时可触及的容器）

## 物品/家具/载具生成

程序最多只会生成到地物、房间这一级。地物上有什么以及房间里有什么就全靠 LLM 填充/瞎编。

瞎编的内容重新落实到游戏状态，靠的是 Game State Router 先定义玩家行为并初步识别行为的限制条件，然后令 Game State Manager 创建所需的物品/家具/载具的 Cardboard 版。

### 示例

0. 玩家发言检查冰箱内容
1. Game State Router: 判断其行为，然后*程序*根据行为种类列举所有需要注意的地方
  - 比如玩家的电工等级、周遭的环境光（TODO更具体的条件）
  - 这部分是给 Game State Manager 用的
2. Game State Manager: 查找*程序*提供的模板后，挑选出最合适的模板与变种，然后附加一段“软性内容的提示(如陈旧的外观、划痕)”。返回的模板与变种结果用*程序*随机生成。
  - 比方说，模板与变种挑选的是 冰箱-豪华冰箱，内容属性挑选的是“中式酱料”+“蔬菜” (最终会变成 Cardboard Loots)
  - *程序*会按照 冰箱-豪华冰箱 随机填充较好的压缩机和外壳 (填充两个 Cardboard Item)
  - 填充了 Cardboard Loots 与两个 Cardboard Item 以及附加了软性提示的这个东西就是 Cardboard Furniture 的冰箱了
  - *程序*还可以根据 Cardboard Loots 与 Cardboard Item 划定质量、长度、体积的范围
3. Book Composer 根据 Cardboard Furniture 冰箱陈述观察结果

如果是搬冰箱，也需要 Cardboard Furniture 冰箱来提供质量、长度、体积等的范围；拆冰箱、把冰箱推倒以阻挡追兵等场景就更合适了。
