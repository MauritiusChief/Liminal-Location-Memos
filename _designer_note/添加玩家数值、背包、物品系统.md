## 物品

物品以建筑id、房间id、容器id或经纬度的形式记录在存档中。

物品的所占空间以包裹换算大小计量，也就是长宽高之总和。

## 背包、容器

容器以建筑id、房间id、其他容器id或经纬度的形式记录在存档中。

玩家的衣服裤兜、手上都可以存放少量物品作为“背包”（本质上是一些玩家随时可触及的容器）

## 物品/家具/载具生成

程序最多只会生成到地物、房间这一级。地物上有什么以及房间里有什么就全靠 LLM 填充/瞎编。

瞎编的内容重新落实到游戏状态，靠的是 Game State Router 先定义玩家行为，然后创建所需的物品/家具/载具（的 Cardboard 版）。创建之后，程序再往其中填充内容物。
填充的也不是细节完备的物品，是一个 Cardboard，需要让当回合 Book Composer 多添油加醋。

等到下一回合，玩家真的和填充物互动了，再让 Game State Manager 填充完备的细节。
